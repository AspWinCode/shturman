# RELEASE DAY 5 QUALITY GATE

Дата: 2026-04-23

## Цель дня

Ввести автоматический контроль качества перед релизом, чтобы ловить регрессии кодировки и сборки до ручного тестирования.

## Что сделано

1. Добавлен новый smoke-тест на кодировку/моджибейк
- Файл: `scripts/smoke-encoding.mjs`
- Проверяет `app`, `components`, `store`, `constants`, `locales`, `api` на характерные маркеры битой UTF-8/cp1251-конверсии.
- Валидация ориентирована на реальные паттерны, без ложных срабатываний на обычные символы (`•`, `—`, `№`).

2. Подключён npm-скрипт
- Файл: `package.json`
- Добавлен: `smoke:encoding`.

3. Прогнан интеграционный web-build
- Команда: `npm run build:web`
- Результат: экспорт web-пакета завершён успешно, `patch-dist.js` применён к 40/40 HTML файлов.

## Проверки

- `npm run smoke:encoding` — passed
- `npm run smoke:content` — passed
- `npm run smoke:api` — passed
- `npx tsc --noEmit` — passed
- `npm run build:web` — passed

## Результат Дня 5

Сформирован стабильный автоматический quality gate для контентной/кодировочной регрессии и подтверждена собираемость web-артефакта. Это снижает риск повторного появления «квакозябр» и скрытых релизных дефектов.
