## 2) P0 (блокеры релиза)

1. Проблемы кодировки (UI-квакозябры)
- Симптом: текст на экранах отображается как `Р...`, `в‚Ѕ` и т.п.
- Критичность: P0 (пользовательский интерфейс фактически поврежден).
- Подтверждено в:
  - `D:\travel_app\app\trip\hotel-select.tsx`
  - `D:\travel_app\api\trainTicketsProvider.js` (локализация данных/строк)

2. Риск дубликатов ключей в списках
- Симптом: `Encountered two children with the same key`.
- Критичность: P0 (нестабильный рендер, предупреждения/непредсказуемое обновление UI).
- Затронутые зоны:
  - транспортные live-офферы
  - train/hotel карточки в result/detail flow

3. Риск навигации до полной готовности layout
- Симптом из истории багов: `Attempted to navigate before mounting the Root Layout component`.
- Критичность: P0 (может блокировать целый экран).
- Зона проверки:
  - root boot flow + ранние redirect/screen guards
  - `D:\travel_app\app\_layout.tsx`
  - `D:\travel_app\app\index.tsx`
  - `D:\travel_app\app\trip\*.tsx`

4. Непредсказуемые live API fallback-сценарии
- Симптом: в ряде мест fallback есть, но UX и тексты не везде консистентны.
- Критичность: P0 (релизная надежность и понятность сценариев при недоступном API).

## 3) P1 (должно быть закрыто до массового запуска)

1. Единообразие CTA и fallback UX
- Для transport/hotels/results нужны унифицированные состояния `loading/error/empty/offline`.

2. Регрессия маршрутов и интересов
- Проверка, что фильтры интересов соблюдаются стабильно (например, архитектура не подменяется музеями без выбранного интереса).

3. Контент и корректность локализации
- Проверка всех строк в auth/onboarding/trip/premium/wallet/profile.

4. Реальные мобильные прогоны
- Android + iOS, минимум по 3 типоразмера экранов, онлайн/оффлайн.

## 4) Freeze на День 1

До закрытия P0:
- Не добавляем новые фичи.
- Не расширяем интеграции API.
- Только стабилизация, исправления и регрессия.

Разрешены:
- Фиксы ошибок и кодировок.
- Доработка fallback/ошибок.
- Улучшение надежности генерации id/keys.
- Тесты и QA-инфраструктура.

## 5) Scope v1.0 (зафиксировано)

В релиз v1.0 включаем:
- onboarding + auth (email/oauth).
- создание поездки.
- выбор отеля.
- выбор транспорта.
- экран результата (маршрут/транспорт/отели/бюджет/карта).
- сохранение поездки.
- wallet (базовый документ-менеджмент).
- premium экран (без обязательной монетизации в первом открытом релизе).

Не включаем в v1.0 (в бэклог после стабилизации):
- крупные новые продуктовые модули и экспериментальные флоу, не покрытые регрессией.
